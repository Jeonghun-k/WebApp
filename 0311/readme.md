# 2. 웹페이지를 만드는 이유

## 기본 개념

웹페이지는 **인터넷에서 사람들이 정보를 보고, 서비스를 사용하고,
거래하고, 소통하도록 만드는 공간**이다.

## 특징

-   웹페이지는 다양한 **멀티미디어 요소**로 구성된다.
    -   텍스트
    -   이미지
    -   오디오
    -   비디오
-   웹페이지 제작에는 **목적(대상 사용자)**이 필요하다.

# 3. 메타데이터 (Metadata)

## 정의

데이터를 설명하는 정보

## 특징

-   데이터의 **속성, 의미, 관계**를 설명
-   일반적으로 **Key-Value 형태**로 저장

## 사용 분야

-   SEO
-   데이터 관리
-   디지털 아카이빙

## 예시

사진을 찍으면 다음 정보가 함께 저장된다.

-   위도 / 경도
-   촬영 날짜
-   노출값
-   조도

이러한 정보들을 **메타데이터**라고 한다.

# 5. JSX (JavaScript XML)

## 정의

JavaScript 안에서 **HTML과 유사한 문법을 사용할 수 있는 문법 확장**

## 특징

-   HTML + JavaScript 결합
-   Babel이 JavaScript로 변환
-   React 요소 생성에 사용

예

    const element = <h1>Hello, JSX!</h1>;

변환 결과

    const element = React.createElement("h1", null, "Hello, JSX!");

------------------------------------------------------------------------

# JSX 특징

## JS 표현식 사용

    <h1>{name}</h1>

## HTML 속성 차이

  HTML    JSX
  ------- -----------
  class   className
  for     htmlFor

------------------------------------------------------------------------

# JSX 장점

-   가독성이 좋음
-   코드가 직관적
-   유지보수 쉬움
-   JavaScript와 자연스럽게 결합
-   조건부 렌더링 쉬움

------------------------------------------------------------------------

# JSX 주의사항

JSX는 **반드시 하나의 부모 태그로 묶어야 한다**

❌ 오류 코드

    function App() {
     return (
       <h1>Hello, JSX!</h1>
       <p>React에서 JSX를 사용합니다.</p>
     );
    }

⭕ 해결 방법

    function App() {
     return (
       <div>
         <h1>Hello, JSX!</h1>
         <p>React에서 JSX를 사용합니다.</p>
       </div>
     );
    }

또는

    function App() {
     return (
       <>
         <h1>Hello, JSX!</h1>
         <p>React에서 JSX를 사용합니다.</p>
       </>
     );
    }
